# 💎 Mellow Code Conventions

Ini adalah aturan baku yang **WAJIB** diikuti oleh AI (Aider) di setiap project yang dikerjakan oleh Senvada.

## 1. Gaya Penulisan Kode (Clean Code)
- **KISS (Keep It Simple, Stupid):** Jangan membuat kode yang terlalu kompleks atau over-engineered jika ada solusi yang lebih sederhana.
- **Penamaan Variabel:** Gunakan nama yang sangat deskriptif. (Misal: `total_user_count` lebih baik daripada `count`).
- **Komentar:** Jangan menulis komentar yang menjelaskan *APA* yang dilakukan kode, tapi jelaskan *KENAPA* kode itu ditulis seperti itu.

## 2. Estetika & UI/UX (Aesthetics First)
- **Premium Design:** Jika membuat file HTML/CSS, hasil akhirnya harus terlihat modern, rapi, dan *wow*. Jangan buat desain "MVP" yang kaku.
- **Animasi:** Tambahkan micro-animations (hover effects, transisi halus) untuk membuat UI terasa hidup.
- **Warna:** Hindari warna dasar (merah terang, biru murni). Gunakan palet warna HSL modern atau dark mode yang elegan.

## 3. Struktur Project
- Tidak boleh membuat file asal-asalan di root directory.
- Pisahkan logika dari tampilan (separation of concerns).
